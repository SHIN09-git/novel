# Novel Director

Novel Director 是一个面向 AI 长篇小说创作的本地桌面 MVP。它不是普通文本编辑器，而是用于管理小说圣经、章节复盘、角色当前状态、伏笔账本、阶段摘要和 Prompt 上下文的“小说导演台”。

## 技术栈

- Electron
- Vite
- React
- TypeScript
- 本地 JSON 持久化

当前版本默认将数据保存到 Electron `userData` 目录下的 `novel-director-data.json`。代码中已保留 Storage / Repository 分层，后续可替换为 SQLite。

## MVP 核心闭环

1. 创建小说项目。
2. 在小说圣经中填写长期稳定设定。
3. 创建主要角色卡和伏笔。
4. 编写第 1-3 章，并填写章节复盘字段。
5. 在阶段摘要页生成 1-3 章阶段摘要草稿。
6. 在 Prompt 构建器中选择第 4 章、模式和上下文模块。
7. 生成结构化 prompt，查看 token 估算，复制或保存版本。

## AI 章节复盘自动化

章节详情页提供 4 个入口：

- 生成章节复盘草稿
- 从正文提取角色变化
- 从正文提取伏笔
- 生成下一章风险提醒

如果设置页没有配置 API Key，系统会生成本地结构化模板，不会崩溃。配置 API Provider、API Key、Base URL、Model Name 后，应用会通过主进程调用 OpenAI/兼容接口，并要求模型返回严格 JSON。

所有 AI 返回内容都会先进入草稿预览：

- 章节复盘字段需要点击“应用”或“应用到章节复盘”后才写入章节。
- 角色状态候选需要点击“应用到角色卡并记录日志”后才更新角色卡并新增 `CharacterStateLog`。
- 新伏笔候选需要点击“加入伏笔账本”后才创建伏笔。
- 已有伏笔推进/回收候选需要点击“应用状态变更”后才修改伏笔状态。
- 如果 AI 返回不是合法 JSON，会显示“解析失败，可手动复制”，并保留原始返回文本，不污染项目数据。

## 章节生产流水线

项目工作区新增“生产流水线”页面，用于半自动生成下一章：

1. 选择目标章节编号、生成模式、预计字数、读者情绪目标和上下文预算。
2. 点击“开始生成”。
3. 系统按步骤执行：上下文预算选择、构建上下文、生成任务书、生成正文、复盘章节、提取角色更新、提取伏笔更新、一致性审稿、质量门禁、等待用户确认。
4. 每一步都会显示状态、输出预览、错误信息，并提供重试和跳过按钮。
5. 正文草稿会保存为 `draft`，点击“接受章节草稿”后才写入章节。
6. 角色、伏笔、章节复盘等长期记忆更新都会先进入候选区，点击接受后才写入项目数据。
7. 一致性审稿报告会保存并展示，供人工检查时间线、设定冲突、角色 OOC、伏笔误用和节奏问题。
8. 质量门禁报告会给出总分、维度分、问题证据、必修项和可选建议；低分草稿仍可强制接受，但需要二次确认。

没有 API Key 时，流水线不会崩溃，会生成本地模板草稿和候选项，方便验证流程。

## 记忆预算调度与质量门禁

Prompt 构建器和生产流水线都支持记忆预算调度。系统会按预算自动选择小说圣经、最近章节、阶段摘要、角色、伏笔和时间线事件，并展示预计 token、被纳入内容、被省略内容和上下文风险。预算较低时会优先压缩文风样例、旧章节详细回顾、低权重伏笔、非相关角色、旧阶段摘要和时间线事件。

生产流水线在一致性审稿后会执行质量门禁。质量门禁会评估剧情连贯、角色一致性、伏笔控制、文风匹配、节奏、情绪兑现、原创性和任务书遵守情况。质量门禁未通过时，接受章节草稿和应用长期记忆候选都会弹出确认，避免低质量内容默认污染角色卡、伏笔账本和章节复盘。

质量门禁问题可以生成修订候选。修订候选会先显示为草稿，用户点击“应用到草稿”后才会替换当前生成草稿正文。

## 数据导入导出

设置页支持导出当前全部数据为 JSON，也支持导入 JSON 覆盖当前本地数据。

## 数据保存路径

设置页的“本地数据”卡片可以查看和修改当前数据文件路径：

- “选择保存位置”可以选择文件夹或 `.json` 文件。选择文件夹时，应用会使用该文件夹下的 `novel-director-data.json`。
- “迁移当前数据到新位置”会先保存当前数据，再写入新路径；如果目标文件已存在，会提示是否覆盖。
- “打开所在文件夹”会在系统文件管理器中定位当前数据文件。
- “恢复默认路径”会把数据迁移回 Electron `userData` 下的默认 `novel-director-data.json`。

数据路径配置保存在独立的 `app-config.json` 中。迁移成功后后续读写都会使用新路径；迁移失败会保留原路径，不会破坏旧数据。

## 章节复制与正文导出

章节页正文编辑器下方提供快捷操作：

- 复制正文
- 复制章节标题 + 正文
- 导出当前章节为 `.txt`
- 导出当前章节为 `.md`
- 批量导出全部章节为合并 `.txt` 或 `.md`

导出文件会通过系统保存对话框选择位置，文件名会自动移除 Windows 不允许的字符。
