# Novel Director

Novel Director 是一个面向 AI 长篇小说创作的本地桌面 MVP。它不是普通文本编辑器，也不是 AI 聊天壳子，而是用于管理小说圣经、章节复盘、角色当前状态、伏笔账本、阶段摘要、Prompt 上下文、章节生产流水线和修订版本的“小说导演台”。

核心目标是减少长篇创作中的上下文漂移、角色状态混乱、伏笔遗忘、token 过高和章节任务不清晰。

## 技术栈

- Electron
- Vite
- React
- TypeScript
- 本地 JSON 持久化

当前版本默认把数据保存到 Electron `userData` 目录下的 `novel-director-data.json`。代码中保留了 Storage / Repository 分层，后续可替换为 SQLite。

## 安装与启动

```bash
npm.cmd install
npm.cmd run dev
```

Windows PowerShell 可能会阻止 `npm.ps1`，建议直接使用 `npm.cmd`。

常用脚本：

```bash
npm.cmd run typecheck
npm.cmd run build
npm.cmd run smoke
npm.cmd run rc:fixture
npm.cmd test
```

`npm.cmd run rc:fixture` 会生成开发用回归数据《雾城测试稿》，位置在 `tmp/rc-regression/novel-director-data.json`，并输出章节导出文件和回归报告。这个文件可以通过设置页的数据导入功能导入应用，也可以作为发布候选测试样本。
`npm.cmd test` 会先生成 RC 回归数据，再运行 `scripts/validate-app-data.mjs` 检查关键集合字段、状态枚举和项目关联完整性。

## 配置 API

进入“设置”页，在 API 区域填写：

- API Provider：OpenAI / Compatible API / Local Model
- API Key
- Base URL
- Model Name
- Temperature
- Max Tokens

没有 API Key 时，AI 相关按钮不会导致应用崩溃，会生成本地结构化模板或给出友好错误。应用不会把 API Key 打印到控制台；远端错误返回会做截断和脱敏。

## 创建小说项目

推荐流程：

1. 在首页创建项目，填写题材、简介、目标读者、核心爽点和整体风格。
2. 进入“小说圣经”，填写世界观、核心命题、主线冲突、文风样例和不可违背设定。
3. 在“角色”页创建主要角色，并维护“当前戏剧状态”。
4. 在“伏笔”页登记伏笔，标记状态、权重和预计回收章节。
5. 在“章节”页写正文，并填写章节复盘字段。
6. 每 3 章在“阶段摘要”页生成阶段摘要草稿。
7. 在 Prompt 构建器或生产流水线中准备下一章。

## 章节生产流水线

“生产流水线”把下一章生成拆成可见步骤：

1. 上下文预算选择
2. 构建上下文
3. 生成任务书
4. 生成正文
5. 章节复盘
6. 角色更新候选
7. 伏笔更新候选
8. 一致性审稿
9. 质量门禁
10. 等待确认

每一步都会保存状态、输出预览和错误信息。正文草稿会保存为 draft，不会直接覆盖正式章节。角色、伏笔、章节复盘等长期记忆更新都会先进入候选区，只有用户点击接受后才会写入项目数据。

生成前，流水线会读取上一章的“章节衔接桥”。如果没有已保存的桥接数据，会使用上一章结尾片段作为临时依据，要求下一章开头承接上一章最后一幕，避免重新开场、重置人物状态或重复介绍已知设定。

质量门禁会给出总分、维度分、问题证据、必修项和可选建议。低分草稿可以强制接受，但需要二次确认。
质量门禁现在额外检查章节连续性和冗余控制：如果开头没有接住上一章钩子、人物身体/情绪状态被重置，或出现重复环境描写、重复解释和抽象强化词，会建议进入修订工作台处理。

## 修订工作台

“修订工作台”用于单章深度修订，而不是批量生成多章：

- 可选择已有章节或生产流水线草稿。
- 支持全文修订，也支持粘贴局部段落进行局部修订。
- 修订类型包括去 AI 味、加强冲突、润色文风、优化对白、压缩节奏、增强情绪、修复 OOC、修复伏笔误用和自定义指令。
- 也支持加强章节衔接、减少冗余、压缩描写、删除重复解释和强化转场承接，用于修复章节像“重新启动”或 AI 味过重的问题。
- 修订结果保存为 RevisionVersion，不会直接覆盖原文。
- 接受修订版本前，系统会把旧正文保存为 ChapterVersion。
- 章节页的“版本历史”可以复制、恢复和删除旧版本。
- 如果章节有质量门禁 issue，可以从修订工作台点击 issue 发起定向修订。

## Prompt 构建器

Prompt 构建器会按轻量 / 标准 / 完整 / 自定义预算选择上下文，并展示：

- 当前 token 预算
- 实际估算 token
- 已纳入章节、角色、伏笔和阶段摘要
- 被省略内容及原因
- 上下文风险提示
- 可编辑的最终 prompt

用户可手动勾选本章相关角色和伏笔，并复制或保存 prompt 版本。
准备第 N 章时，Prompt 构建器会优先纳入第 N-1 章到第 N 章的“上一章结尾衔接”模块；用户也可以在“章节衔接”面板里补充本章开头必须承接和禁止重置的指令。

## 章节衔接与冗余压缩

章节页的复盘区域提供“下一章衔接状态”卡片，可以记录上一章结尾位置、身体状态、情绪状态、未完成动作、下一章必须接住的第一拍、禁止重置项和开放小张力。

AI 章节复盘会把这些内容作为候选建议展示，用户确认后才保存为 `ChapterContinuityBridge`。这不会自动污染长期记忆。

生产流水线生成正文后会创建轻量 `RedundancyReport`，记录重复词组、重复环境描写、重复解释、过度强化词和压缩建议。Run Trace 会记录本次使用的是保存的衔接桥、上一章结尾兜底还是手动指令，并关联冗余报告，方便排查章节写歪的原因。

## 导出正文

章节页提供：

- 复制正文
- 复制章节标题 + 正文
- 导出当前章节为 `.txt`
- 导出当前章节为 `.md`
- 批量导出全部章节为合并 `.txt` 或 `.md`

文件保存通过 Electron 主进程的系统保存对话框完成，renderer 不直接访问 Node `fs`。文件名会移除 Windows 不允许的字符。

## 本地数据路径

设置页的“本地数据”卡片可以：

- 查看当前数据文件路径
- 选择新的保存位置
- 迁移当前数据到新位置
- 打开数据文件所在文件夹
- 恢复默认路径

数据路径配置保存到 Electron `userData` 下的 `app-config.json`。如果选择文件夹，应用会使用该文件夹下的 `novel-director-data.json`；如果选择 `.json` 文件，则直接使用该文件。

迁移数据时会先保存当前数据；如果目标文件已存在，会提示覆盖或取消。原路径会保留备份。JSON 数据写入采用临时文件 + `.bak` 备份 + rename 的原子写入策略。读取 JSON 失败时会把原文件备份为 `.corrupt.<timestamp>.json`，再回退为空数据，避免直接覆盖损坏文件。

## Release Candidate 回归数据

运行：

```bash
npm.cmd run rc:fixture
```

脚本会生成一个完整测试项目《雾城测试稿》，覆盖：

- 小说圣经
- 3 个主要角色
- 5 条伏笔，包含低权重、高权重、payoff 和预计第 5 章回收
- 第 1-4 章
- 1-3 章阶段摘要
- 第 4 章 prompt 版本
- 第 4 章生产流水线记录
- 章节复盘、角色更新、伏笔更新候选
- 质量门禁报告
- 修订会话、修订版本和章节历史版本
- 当前章节导出和全章节导出
- 模拟数据路径迁移与 JSON 损坏备份

输出报告位于 `tmp/rc-regression/rc-regression-report.json`。

## 常见问题

### 打开应用是一片空白怎么办？

先运行：

```bash
npm.cmd run typecheck
npm.cmd run build
```

如果构建通过，再运行 `npm.cmd run dev`。仍然空白时，可检查终端是否有 renderer 报错，或尝试恢复默认数据路径。

### 没有 API Key 能用吗？

可以。项目管理、小说圣经、章节、角色、伏笔、阶段摘要、Prompt 构建、复制导出、版本历史都可以本地使用。AI 自动复盘、生产流水线和修订会返回本地模板或友好错误，不会崩溃。

### AI 返回解析失败怎么办？

应用会保留原始返回文本，不会写入长期记忆。你可以手动复制原文，或调整模型、Max Tokens、Base URL 后重试。

### 生产流水线生成的内容会直接污染角色卡和伏笔账本吗？

不会。正文草稿需要用户接受才写入章节；角色、伏笔、复盘等长期记忆都先进入候选区，确认后才写入。

### 修订会覆盖原文吗？

不会直接覆盖。修订先生成版本；接受版本时会自动保存旧正文到章节版本历史。

### 数据文件在哪里？

默认在 Electron `userData` 目录下的 `novel-director-data.json`。具体路径可以在设置页“本地数据”区域查看，也可以改到自定义位置。
